Titan is a family of neural architectures that combines short-term memory (via attention) and long-term memory (via a neural memory module) to address the limitations of traditional recurrent models and attention mechanisms. It is specifically designed to handle long-context tasks effectively while maintaining fast and parallelizable training and inference.

Limitations of Traditional Sequence Models:

Computational Complexity

  • Transformers: Transformers are attention-based models that excel in sequence modeling tasks by learning from context and scaling efficiently. Attention modules function as associative memory, retrieving information based on query-key similarities. However, they are computationally expensive, with time and memory requirements growing quadratically with context length. Real-world tasks like language modeling, video understanding, and time series forecasting often involve very large contexts.

  • Linear Transformers: Proposed as a solution, they replace the softmax attention mechanism with kernel functions, significantly reducing memory usage. While more efficient and better suited for longer contexts, they compress data into matrix-valued states, limiting their performance compared to standard Transformers. Linear models handle very long contexts efficiently but struggle to compress context effectively into smaller representations.

Memory

Memory plays a crucial role in learning and decision-making, as it allows systems to store, update, and retrieve information based on inputs.

  • RNN: RNNs (Recurrent Neural Networks) use a vector-based memory where information is updated and retrieved based on input sequences.

  • Transformers & Linear Transformers: Transformers maintain memory by adding key-value pairs, which can be retrieved by comparing queries with keys. Unlike RNNs, Transformers retain all historical data, while linear Transformers compress this data into matrices for efficiency. Storing memory in a simplified form (like a single vector or matrix) might not be sufficient for long-term memory storage, and a deeper, more advanced memory structure may be needed to handle complex memory storage requirements.

Specialized and interconnected memory components that can effectively store, update, and retrieve information across short-term and long-term contexts can handle large, complex datasets over extended periods.

Solution: Long Term Memory

The authors came up with a neural long-term memory module designed to address the lack of long-term memory in models, enabling them to memorize, forget, and retrieve information during the test phase. The memory module learns when to memorize and forget at test time, improving generalization.

Long-Term Memory Module

To create a neural long-term memory module, we need a model that encodes past history into its parameters. Large language models (LLMs) are known to memorize their training data, but memorization is usually undesirable because it limits generalization and raises privacy concerns. Simply memorizing training data can lead to poor performance when the data is out-of-distribution at test time. Instead, they propose an online meta-model that learns when to memorize and forget the data at test time, allowing the model to generalize better.

Solution: Persistent Memory

In neural networks, especially in models like Transformers, memory systems are typically contextual, meaning they depend on the input sequence. However, for effective learning, the model also requires input-independent parameters that store task-specific knowledge. This persistent knowledge helps guide the model when processing different tasks, allowing it to generalize and perform better by capturing overarching task-related information, independent of any single input. Without this, the model might struggle to process tasks that require a broader understanding beyond the immediate input.

Persistent memory (or meta-memory) refers to a set of learnable parameters that are input-independent and store general knowledge about the task. These parameters are appended at the start of the input sequence, enabling the model to retain and apply task-relevant knowledge throughout its operations.

The persistent memory is represented by the learnable parameters:

Where â€œâˆ¥\parallelâˆ¥â€ denotes concatenation of the parameters with the actual input sequence.

This form of memory ensures that task-related context is consistently available at the start of each sequence, contributing to more efficient processing and reduced biases in attention mechanisms.

The memory is structured to capture short-term, long-term, and persistent information, mimicking how humans process and retain information. Choosing an Architecture

  • MAC: Ideal when the model needs dynamic interaction between current and past contexts.

  • MAG: Suitable for scenarios requiring a balance between precise short-term and fading long-term memory.

  • MAL: Best for simpler setups or when memory needs to be treated as a standalone processing step.
